# บรีฟสำหรับ Cowork — สร้างไฟล์ข้อสอบ JSON (สำหรับปุ่มนำเข้าใน RxTU10)

ส่งไฟล์นี้ให้ cowork ทั้งไฟล์ หรือก็อปบล็อก "คำสั่ง" ด้านล่างไปสั่งได้เลย
ทีมวิชาการเอาผลลัพธ์ไปวางในแท็บ **ข้อสอบ → 📥 นำเข้าข้อสอบหลายข้อ (JSON)** แล้วกดนำเข้า

---

## รูปแบบที่ต้องส่ง

**JSON array** ของ object — ตรง schema ของ collection `questions`:

```json
[
  {
    "question": "ยาใดเป็น first-line ในการรักษา community-acquired pneumonia ในผู้ใหญ่สุขภาพดี",
    "choices": ["Amoxicillin", "Vancomycin", "Meropenem", "Colistin"],
    "answer": 0,
    "category": "ยาปฏิชีวนะ",
    "explanation": "Amoxicillin เป็น first-line ตามแนวทาง CAP ผู้ป่วยนอกที่ไม่มีปัจจัยเสี่ยง"
  },
  {
    "question": "Paracetamol ออกฤทธิ์ยับยั้งเอนไซม์ใดเป็นหลัก",
    "choices": ["COX ในระบบประสาทส่วนกลาง", "COX-1 ในกระเพาะ", "Lipoxygenase", "Phospholipase A2"],
    "answer": 0,
    "category": "เภสัชวิทยา"
  }
]
```

## กติกา (ทำตามเป๊ะ ไม่งั้นข้อนั้นถูกข้าม)

| field | บังคับ? | กติกา |
|---|---|---|
| `question` | ✅ | ข้อความโจทย์ ห้ามว่าง · ยาวไม่เกิน 500 ตัวอักษร (เกินถูกตัด) |
| `choices` | ✅ | **array** ของ string · ตัวที่ไม่ว่าง **2–6 ตัว** · แต่ละตัวไม่เกิน 200 ตัวอักษร |
| `answer` | ✅ | **ลำดับตัวเลือกที่ถูก เริ่มนับที่ 0** (ตัวแรก = 0, ตัวสอง = 1 ...) ถ้าเกินช่วงจะถูกตั้งเป็น 0 |
| `category` | ไม่ | หมวด/กลุ่มเนื้อหา ไม่เกิน 60 ตัวอักษร |
| `explanation` | ไม่ | คำอธิบายเฉลย ไม่เกิน 1000 ตัวอักษร |

**ห้ามใส่ field อื่น** เช่น `isPublished`, `id`, `createdAt` — ระบบจัดการเอง (ทุกข้อนำเข้าเป็น "ร่าง" เสมอ
ทีมวิชาการตรวจแล้วค่อยกดเผยแพร่ทีละข้อ)

## พลาดบ่อย — ห้ามทำ

1. ⚠️ **ห้ามใช้ smart quotes** (`" "` `' '` แบบ Word/มือถือ) — ทำให้ทั้งไฟล์ parse ไม่ผ่าน ใช้ `"` ตรงๆ เท่านั้น
2. ⚠️ **ห้ามมี trailing comma** (จุลภาคหลัง element สุดท้ายก่อน `]` หรือ `}`)
3. ⚠️ `answer` เริ่มที่ **0** ไม่ใช่ 1
4. อย่าใส่เฉลยเป็นข้อความ (`"answer": "ก"`) — ต้องเป็น **เลข index**
5. ผลลัพธ์ต้องเป็น **JSON ล้วน** ไม่มี markdown ```` ``` ```` ครอบ ไม่มีคำอธิบายนอก array

## ข้อแนะนำปริมาณมาก

- นำเข้าได้ครั้งละหลายร้อยข้อ (ระบบ chunk ให้อัตโนมัติ) แต่**แบ่งไฟล์ตามหมวด**จะ track และตรวจง่ายกว่า
- เริ่มจากไฟล์เล็ก **2–3 ข้อก่อน** ให้ผ่าน flow แล้วค่อย scale

## บล็อก "คำสั่ง" (ก็อปไปสั่ง cowork ได้เลย)

> ช่วยสร้างคลังข้อสอบเภสัชศาสตร์เป็น **JSON array** ตาม schema นี้:
> แต่ละข้อ = `{ "question": string, "choices": [string,...], "answer": number, "category"?: string, "explanation"?: string }`
> กติกา: `choices` ที่ไม่ว่าง 2–6 ตัว · `answer` = index ของตัวเลือกที่ถูก **เริ่มนับที่ 0** ·
> ห้าม smart quotes ห้าม trailing comma · ตอบเป็น **JSON ล้วน** ไม่ครอบ markdown ไม่มีข้อความอื่น ·
> หมวด `category` ให้สอดคล้องกัน (เช่น "ยาปฏิชีวนะ", "เภสัชวิทยา", "เภสัชจลนศาสตร์") ·
> ใส่ `explanation` สั้นๆ อธิบายว่าทำไมข้อนั้นถูก
> เริ่มที่หมวด ___ จำนวน ___ ข้อ
