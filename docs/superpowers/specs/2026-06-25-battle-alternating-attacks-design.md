# Battle: สลับฝั่งตี (Alternating Attacks, HS Battlegrounds style)

วันที่: 2026-06-25 · สถานะ: อนุมัติ design แล้ว รอเขียน plan

## ปัญหา / เป้าหมาย

ระบบ battle ปัจจุบัน (`src/utils/battleEngine.js`) เป็นแบบ **round-based พร้อมกัน**: ทุกรอบ
ตัวที่ยังไม่ตายของทั้งสองฝั่งได้ตี 1 ที โดย **สุ่มลำดับ** (กัน first-mover) และ **สุ่มเป้า**
→ ตำแหน่งซ้าย-ขวาในทีมไม่มีผลเชิงกลยุทธ์เลย

ต้องการเปลี่ยนเป็นแบบ **Hearthstone Battlegrounds**:
- สลับฝั่งตี **ทีต่อที** (A, B, A, B, …)
- แต่ละฝั่งเลือกตัวออกตี **จากซ้ายไปขวา วนกลับ**
- **สลับฝั่งเสมอ ไม่ว่าใครเหลือกี่ตัว** — ฝั่งที่เหลือ 1 ตัว ตัวนั้นได้ตีทุกครั้งที่ถึงตา
  (เช่น A เหลือ 1 ตัว vs B 4 ตัว → ลำดับ A,B,A,B,… ตัวเดียวของ A ได้สลับตีทุกที)

## การตัดสินใจ (ยืนยันกับผู้ใช้แล้ว)

1. **ใครตีก่อน:** ฝั่งที่มีตัว *ยังไม่ตาย* เยอะกว่าตีก่อน · เท่ากัน → สุ่ม (ตรงตาม HS BG)
2. **การเลือกเป้า:** คงเดิม = **สุ่มเป้า** ในฝั่งตรงข้าม (HS BG ก็สุ่ม ยกเว้น Taunt ซึ่งเกมนี้ไม่มี)
   ลำดับซ้าย→ขวา มีผลแค่ "ใครออกตี" ไม่ใช่ "ใครโดนตี"

## ขอบเขต

แก้เฉพาะ **ลำดับการออกตี** ในลูปหลักของ `simulateBattle()` กลไก damage ทั้งหมด
(สุ่มเป้า `pick`, ตัวคูณธาตุ `elementMult`, crit, variance ใน `hit`) และ **format ของ
`attack` event คงเดิมทุกฟิลด์** → downstream ไม่ต้องแก้ logic:
- `src/utils/battleStats.js` — อ่าน `attack` event (side/attacker/target/dmg/dead) เหมือนเดิม
- `src/components/battle/BattleReplay.vue` — dispatch ตาม `event.t` (`round`/`attack`/`end`) เหมือนเดิม

## อัลกอริทึมใหม่ (แทนลูป round เดิม บรรทัด ~45–55)

ตัวแปรสถานะ: แต่ละฝั่งมี cursor `nextIdx` (เริ่ม 0 = ซ้ายสุด)

```
หาตัวออกตีของฝั่ง side:
  ไล่จาก nextIdx ไปขวา วนกลับมาซ้าย (mod ขนาดทีม) หาตัวแรกที่ hp > 0
  ถ้าไม่มีตัวเหลือ → null (ฝั่งนี้แพ้)

ลูปหลัก:
  1. กำหนดฝั่งเริ่ม first:
       countAlive(A) > countAlive(B) → 'A'
       countAlive(B) > countAlive(A) → 'B'
       เท่ากัน → rand() < 0.5 ? 'A' : 'B'
  2. cur = first; turn = 0
  3. while (มีทั้งสองฝั่งยังไม่ตาย && turn < cap):
       - attacker = หาตัวออกตีของ cur
       - ถ้า attacker == null → break (cur แพ้)
       - hit(attacker, foes ของ cur)        // ใช้ hit() เดิม สุ่มเป้า
       - เลื่อน cursor ฝั่ง cur ไป (attackerIdx + 1) mod size
       - cur = อีกฝั่ง   // สลับเสมอ
       - turn++
  4. ตัดสินผลด้วย pct()/alive() เดิม
```

### Round marker (เพื่อ replay "รอบ N")

นับ "รอบ" = ทุกครั้งที่ control วนกลับมาถึง **ฝั่งที่เริ่มก่อน (first)** = ครบ 1 รอบ ping-pong
emit `{ t: 'round', n }` ก่อนเริ่มแต่ละรอบ (รอบแรก n=1) — รูปแบบเดิม UI/replay ไม่ต้องแก้

### Safety cap

damage = `max(0, atk * m)` โดย atk > 0 และ m > 0 เสมอ → HP ลดทุกการตี → battle จบแน่นอน
ไม่ลูปนิรันดร์ แต่ยังคง cap กันเหนียว: **คงค่า `BATTLE_CFG.maxRounds` เดิม (30) ในความหมายใหม่
= จำนวนรอบ ping-pong สูงสุด** (1 รอบ = first ตี 1 ที + อีกฝั่งตี 1 ที) — ถ้าชน cap ตัดสินด้วย hpPct เดิม

## ผลข้างเคียงที่ตั้งใจ

- ลำดับใน `TeamPicker` เริ่มมีผล: ตัวซ้ายสุดได้ออกตีก่อนและบ่อยกว่าเมื่อตัวตาย → จัดทีมมีกลยุทธ์
- เป้ายังสุ่ม จึงไม่มี "แท็งก์หน้า" (ไม่มี taunt) — ลำดับมีผลเฉพาะ "ใครได้ตี" เท่านั้น

## เทสที่ต้องอัปเดต (`src/utils/battleEngine.test.js`)

คงได้ (ไม่ต้องแก้):
- deterministic: seed เดิม → ผลเหมือนเป๊ะ
- log จบด้วย end event, winner ตรง
- ธาตุได้เปรียบ/เกรดสูงกว่า ชนะเกินครึ่ง
- ทีมว่างฝั่งหนึ่ง → อีกฝั่งชนะ
- attack event มี eff ตรง matchup ธาตุ (fist vs scissors ยังคงทุกตี super/weak)

ต้องแก้/เพิ่ม:
- เทส round marker — ความหมายเปลี่ยน (รอบ = ping-pong cycle) ปรับ assertion ให้ตรงกลไกใหม่
- เพิ่ม: **ฝั่งตัวเยอะกว่าได้ตีก่อน** (เทียบ attack event ตัวแรกของ log)
- เพิ่ม: **สลับฝั่งจริง** — เมื่อทั้งสองฝั่งมีตัวเท่ากันและยังไม่ตาย ลำดับ side ของ attack event สลับ A/B/A/B
- เพิ่ม: **1 ตัว vs หลายตัว** — ฝั่งที่เหลือ 1 ตัวได้เป็น attacker ทุกครั้งที่ถึงตา (ตัวเดิมซ้ำ)
