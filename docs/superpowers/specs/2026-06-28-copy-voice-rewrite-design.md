# Copy/Voice Rewrite — Design

วันที่: 2026-06-28
สถานะ: รอ user review

## เป้าหมาย

ปรับ "เสียง" (voice) ของข้อความทั่วเว็บ RxTU10 ให้:
- **เป็นกันเอง** — พูดกับผู้เล่นแบบเพื่อน ไม่เป็นทางการเกินไป
- **ตอบสนองวัตถุประสงค์** — บอกชัดว่าฟีเจอร์นี้ทำอะไรได้ ผู้เล่นควรทำอะไรต่อ
- **อธิบายฟังก์ชันให้เข้าใจง่าย** — ภาษาที่นักศึกษาเภสัชอ่านแล้วเก็ตทันที
- **ไม่มีคำหวือหวา/โอ้อวด** — ตัดคำแบบ "ความเก๋า", "เทพ", "ปัง", "ไม่มีใครรอด"

ไม่ใช่การเขียนใหม่ทั้งหมดให้จืด — flavor ที่ให้ความน่ารัก/ยินดีกับผู้เล่นยังเก็บไว้ แต่ต้องอิงสิ่งที่ผู้เล่นทำจริง ไม่เคลมเกินจริง

## ขอบเขตที่ไม่แตะ (สำคัญ)

- ไม่แก้ logic / ตัวเลขเศรษฐกิจ / ชื่อฟิลด์ schema / ชื่อ route / id ใด ๆ
- ไม่แตะ data identity: ชื่อเพ็ท, ชื่อพืช, ชื่อโรค/ยา, ชื่อหมวด
- ไม่แตะการ render emoji (`<Emoji>` / fetch-fluent) — แต่ **ห้ามเพิ่ม emoji ใหม่ลง JS string** (toast/ternary) เพราะ tofu; ถ้าจำเป็นต้องผ่าน `<Emoji>` ใน template
- ไม่ทำ i18n / ไม่เพิ่ม dependency
- งานนี้เป็น copy ล้วน → verify ด้วย `npm run build` เขียว + eyeball ไม่มีการเปลี่ยนพฤติกรรม

## หลักการเสียง (Voice Guide — 6 ข้อ)

1. **พูดกับผู้เล่นตรง ๆ ว่า "คุณ"** — สุภาพแต่เป็นกันเอง (คงสรรพนามเดิมที่ใช้ทั้งแอป)
2. **บอกก่อนว่า "นี่คืออะไร ทำอะไรได้"** แล้วค่อยบอกวิธีใช้/เงื่อนไข
3. **ตัดคำโอ้อวด/หวือหวา** — เก๋า, เทพ, เซียน(ในคำอธิบาย), สุดยอด, ปัง, อลังการ, จัดเต็ม, "ไม่มีใครรอด" ฯลฯ
4. **ใช้คำที่เข้าใจทันที** เลี่ยงศัพท์เกม/อังกฤษลอย ๆ ที่ไม่อธิบาย — ถ้าต้องใช้ ให้ขยายความครั้งแรกที่เจอ
5. **สั้น กระชับ หนึ่งความคิดต่อประโยค**
6. **flavor เก็บความอบอุ่น/ยินดีได้ แต่อิงสิ่งที่ผู้เล่นทำจริง ไม่เคลมเกินจริง**

### Glossary — คำเลิกใช้ → แนวทางแทน

| เลิกใช้ | แทนด้วย |
|---|---|
| เก๋า, ปัง, อลังการ, จัดเต็ม, สุดยอด | ตัดทิ้ง หรือบอกผลลัพธ์จริง ("รายได้ยิ่งเยอะ", "เล่นมาถึงไหน") |
| เซียน/เทพ ในคำอธิบาย | "เก่ง / ขยัน / ชำนาญ" (คงได้เฉพาะใน achievement title สนุก ๆ) |
| เคลมเว่อร์: "ไม่มีตัวไหนรอด", "ต้องหลบ", "ทั้งรุ่นต้องยอม" | อิงสิ่งที่ผู้เล่นทำจริง |
| ศัพท์เกม/อังกฤษที่ไม่อธิบาย | คงได้ แต่ขยายความครั้งแรกที่เจอ |

### ตัวอย่าง before → after

| จุด | เดิม | ใหม่ |
|---|---|---|
| guide บ้าน | "...เป็นเครื่องวัดความเก๋าของคุณ" | "เลเวลบ้านจะโชว์บนโปรไฟล์ให้เพื่อนเห็นว่าคุณเล่นมาถึงไหนแล้ว" |
| achievement quiz_100 | "ข้อสอบเห็นแล้วต้องหลบ" | "ทำข้อสอบมา 100 ข้อ ขยันสุด ๆ" |
| achievement pet_all | "ครบทุกชนิด ไม่มีตัวไหนรอด!" | "สะสมครบทุกชนิดแล้ว เก่งมาก" |
| MigrationWelcome | "ยิ่งสูงยิ่งเก๋า!" | "ยิ่งอัปสูง รายได้ต่อวันยิ่งเยอะ" |

## ขอบเขตไฟล์ (copy surfaces)

เรียงตามคนเห็นบ่อย:

| กลุ่ม | ไฟล์ | ระดับการแก้ |
|---|---|---|
| 1. ไกด์ปุ่ม ? | `data/guide.js` (12 หัวข้อ) | เต็ม |
| 2. หน้าจอนักศึกษา | `views/`: Home, Play, Study, Quiz, Shop, Pets, Tower, Arena, Expedition, Me, Members + components (การ์ด Home, `components/shop/LabTab.vue`, WelcomeBox) | เต็ม — หัวข้อ/hint/ปุ่ม/empty state |
| 3. flavor/บุคลิก | `data/achievements.js` (17 flavor), `data/index.js` (เพ็ท flavor ~27) | เก็บ title สนุก + lore เพ็ท · แก้เฉพาะ flavor ที่เคลมเว่อร์ |
| 4. toast นักศึกษา | composables: useFarm, useExpedition, useArena, useDaily, useResidence, useTower + views: Quiz, Study, Shop, Me + components: LabTab, PetDetailModal, MailboxCard | เต็ม |
| 5. onboarding/consent | `components/onboarding/`: ConsentGate, OnboardingWizard, GuestPendingScreen, MigrationWelcome | เต็ม (จุดแรกที่ผู้ใช้เจอ) |
| — Admin/Questions | `views/AdminView.vue`, `views/QuestionsView.vue` + comments component | **light-touch** — แก้เฉพาะที่กำกวมจริง ไม่กวาดทั้งหมด (เห็นเฉพาะทีมงาน) |

### หมายเหตุการตัดสินใจ (เคาะกับ user แล้ว)
- Admin/Questions = light-touch (ไม่กวาดเต็ม)
- achievement title เก็บไว้ทั้งหมด (สนุกแต่ไม่เคลม) — แก้เฉพาะ flavor บรรทัดอธิบาย
- pet flavor ใน `data/index.js` ส่วนใหญ่เป็น lore เชิงบรรยาย (เช่น "ราชาแห่งเวหา...") → เก็บไว้ แก้เฉพาะตัวที่หลุดโทนหวือหวา/เคลม

## Deliverable เสริม: voice guide ถาวร

นอกจากแก้ copy แล้ว สร้าง `docs/voice-guide.md` (= หลักการ 6 ข้อ + glossary + ตัวอย่าง ด้านบน) เป็นเอกสารอ้างอิงถาวร เพื่อให้ copy ของฟีเจอร์ใหม่ในอนาคตคงโทนเดียวกัน · เพิ่มบรรทัดชี้ไปที่ไฟล์นี้ใน `CLAUDE.md` (ส่วน "สไตล์")

## แผนการทำงาน (สำหรับ implementation plan)

จัดเป็น task ตามกลุ่มไฟล์ด้านบน (กลุ่ม 1 → 5 + light-touch admin + voice-guide.md) เพื่อรีวิวทีละก้อน · แต่ละ task:

- จบด้วย `npm run build` เขียว
