# Battle UI Clarity — telegraph + melee พุ่งสุดตัว + เลขดาเมจชัด + modal สรุปผล

**วันที่:** 2026-07-03 · **สถานะ:** อนุมัติดีไซน์แล้ว (รอเขียน plan)
**ที่มา:** backlog 29 มิ.ย. ข้อ 2 — "ปรับ UI หน้า battle (อ่านความชัด + telegraph + ranged + หน้าสรุปแยก)"

## เป้าหมาย

ทำให้ replay การต่อสู้ **อ่านรู้เรื่อง**: เห็นล่วงหน้าว่าตัวไหนกำลังจะตี เห็นชัดว่าตีใคร
เลขดาเมจอ่านออก และหน้าสรุปผลไม่ดันสนามจนต้องเลื่อนยาว

## ขอบเขต

- **แก้ไฟล์เดียว:** `src/components/battle/BattleReplay.vue` (template + script + scoped style)
- **ไม่แตะ engine** (`utils/battleEngine.js`) และไม่แตะ log format — งานอนิเมชัน/เลย์เอาต์ล้วน
- ได้ผลทุกจุดที่ใช้ BattleReplay: หอคอย (TowerView) + สนามประลอง (ArenaView)
- **ไม่ทำในรอบนี้:** เพิ่มจำนวนเพ็ท ranged (data), passive จริง (§5.5 master plan)

## ข้อเท็จจริงจากโค้ดปัจจุบัน (สำรวจ 3 ก.ค.)

- ranged มีอยู่แล้ว 8 ตัว (`atkStyle:'ranged'` + projectile ใน data/index.js) ยิงข้ามสนามได้
- melee ถูกสุ่มเป้าได้ทุกตัวอยู่แล้ว (engine สุ่มเป้า) — แต่อนิเมชันพุ่งแค่ 55% ของระยะใน 150ms
  จึง **ดูเหมือน** ตีอยู่กับที่/ตีแค่ตัวใกล้ → user ขอ "ตีไกล" จริงๆ คือปัญหา visual ไม่ใช่ logic
- ไม่มี telegraph: คลาส `.acting` ขึ้นพร้อมกับตีเลย
- เลขดาเมจ 0.9rem (crit 1.2rem) ลอย 24px ใน 0.6 วิ — เล็ก/หายเร็ว
- หน้าสรุปต่อท้ายใน `.br-ctrl` ใต้สนาม กล่องเดียวกัน สูงจนต้อง overlay เลื่อน

## ดีไซน์ (แนวทาง A — ขยาย state machine เดิม ไม่รื้อ)

คงลูป `step()` + setTimeout chain เดิม เพิ่มเฟสใน `applyAttack` + ปรับ `REPLAY_CFG`
ทุก timing หารด้วย `speed` (×1/×2/×4) เหมือนเดิม

### 1. Telegraph (เงื้อก่อนตี)

- ก่อนเริ่ม motion: ตัวโจมตีขึ้นคลาส `.windup` — ลอยขึ้น + ขอบ/เงาเรืองแสงเหลือง
  + เอนถอยหลังเล็กน้อย (translate ทิศตรงข้ามเป้า ~6-8px)
- ค้าง `windupMs: 250` (ที่ ×1) แล้วค่อยเข้าเฟสพุ่ง/ยิง — ใช้ทั้ง melee และ ranged
- ลำดับต่อ 1 การตี: windup 250ms → motion (lunge/projectile) → impact → คั่น `baseDelay` เดิม

### 2. Melee พุ่งสุดตัว (Hearthstone-style)

- เปลี่ยนระยะพุ่งจาก 55% → **100%** ถึงจุดศูนย์กลางเป้า (ชนทับ) + scale ขยายตอนชน
- `lungeMs` เพิ่มเป็น ~250ms ขาไป, เด้งกลับที่เดิมหลัง impact
- ตัวที่กำลังพุ่งต้อง z-index สูงสุด ไม่โดนการ์ด/แถวอื่นบัง

### 3. Impact + เลขดาเมจ

- ตอน impact: เป้ากระพริบสว่าง (flash ขาว) + เขย่าแรงขึ้นกว่าเดิม
- เลขดาเมจ: ปกติ **1.5rem**, crit **2rem** + text-stroke/เงาเข้ม อ่านออกทุกพื้นหลัง
- ลอยขึ้น ~40px แสดง ~0.9 วิ + offset แนวนอนสุ่มเล็กน้อย (กันเลขหลายป็อปซ้อนทับ)
- callout แพ้ทาง/ต้านทาน (`br-call`) คงพฤติกรรมเดิม

### 4. หน้าสรุปเป็น modal แยก

- ตีจบ → เว้นจังหวะสั้น (~0.5 วิ ให้เห็นสนามจบ) → เปิด modal สรุปทับสนามที่มืดลง
- เนื้อหาเดิมทั้งหมดย้ายเข้า modal: ผลแพ้ชนะ + rewardText + ตาราง 2 ทีม (ดาเมจทำ/รับ, MVP
  กรอบทอง=ทีมชนะ/ม่วง=ทีมแพ้, ตัวตายจาง)
- ปุ่ม **"ดูสนาม"**: ซ่อน modal ชั่วคราว → มีปุ่มลอย "ดูสรุป" กดกลับมาเปิด modal ต่อ
- ปุ่ม **"ปิด"**: emit `close` ออกจาก replay (พฤติกรรมเดิม)
- modal เลื่อนในตัวเองได้ (max-height + overflow-y) — คงการแก้ bottom-nav/safe-area เดิมของ
  `.br-ov.result` ไว้ในรูปแบบใหม่
- แตะนอก modal = เท่ากับกด "ดูสนาม" (peek ไม่ใช่ปิดทิ้ง กันกดพลาดเสียหน้าสรุป)

### 5. งบเวลา (ที่ ×1)

| เฟส | เดิม | ใหม่ |
|---|---|---|
| windup | — | 250ms |
| melee lunge / projectile | 150 / 280ms | 250 / 280ms |
| คั่นจังหวะ (baseDelay) | 380ms | 380ms (คงเดิม) |

รวมช้าลง ~350ms/การตี — ยอมรับได้ตามที่ user เคาะ (telegraph สั้น 0.25 วิ, เร่ง ×2/×4 ชดเชย)

## สิ่งที่ต้องไม่พัง (regression guard)

- ปุ่ม พัก/เล่น · เร่ง ×1/×2/×4 · ข้ามไปผล — ทำงานถูกทุกเฟส (windup ค้างอยู่ต้องหยุด/ข้ามได้)
- `skipToEnd` ต้องล้าง windup/projectile/pop ค้าง แล้วเปิด modal สรุปทันที
- แตะตัว = pause + inspect popover ยังใช้ได้ (แตะระหว่าง windup ด้วย)
- intro READY?→GO! + replay ใหม่ (`reset()`) ล้าง state ครบ ไม่มี timer รั่ว
- emoji ทุกตัวผ่าน `<Emoji>` (ห้าม emoji ดิบ — tofu บนบางเครื่อง)
